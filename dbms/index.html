The idea behind storing forms to files is nice

It looks something like this:

#include <stdlib.h>

struct person {
    int id;
    char fname[20];
    char lname[20];
};

int main() {
  File *file
  /* do some validation */
  ...
  file = fopen("people.bin", "w"); /* open for writing */
  /* do some validation */
  ...

  struct person first_rec  = {1, "Ido", "Kanner"};
  struct person second_rec = {2, "John", "Doe"};

  fwrite(&first_rec, sizeof(struct person), 1, file);
  fwrite(&second_rec, sizeof(struct person), 1, file);

  /* do some validation */
  ...

  fclose(file);

  return 0;
}

But there are issues with this way of storing data.

  • You are managing everything yourself
  • The program will have a lot of boilerplate code
  • For each structure you need a file!
  • Prune to race condition
  • No duplication validation!
  • The structs will not be the same on all platforms
  • Can you think on more issues?

Formally, a "database" refers to a set of related data and the way it is organized. Access to this data is usually provided by a "database management system" (DBMS) consisting of an integrated set of computer software that allows users to interact with one or more databases and provides access to all of the data contained in the database (although restrictions may exist that limit access to particular data). The DBMS provides various functions that allow entry, storage and retrieval of large quantities of information and provides ways to manage how that information is organized.
Source: https://en.wikipedia.org/wiki/Database
